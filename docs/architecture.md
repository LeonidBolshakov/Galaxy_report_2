# Архитектура проекта

Проект — настольное приложение на PyQt6 для подготовки значений к ежемесячному
отчёту в корпорацию.

Основная идея архитектуры: интерфейс, расчёты, форматирование и проверка ввода
разделены по небольшим модулям.

## Основные модули

### `main.py`

Точка входа приложения.

Создаёт `QApplication`, создаёт главное окно `Report` и запускает цикл событий
Qt.

### `report.py`

Главный модуль приложения.

`Report` отвечает за:

- загрузку интерфейса из `_internal/report.ui`;
- хранение текущих финансовых значений;
- подключение сигналов полей ввода;
- расчёты;
- вывод рассчитанных значений в форму;
- динамическое добавление строк платежей;
- копирование результата из полей вывода;
- подтверждение изменения справочных значений.

В этом классе сосредоточен пользовательский сценарий: пользователь вводит
данные, приложение рассчитывает результаты и показывает текст для переноса в
ежемесячный отчёт.

### `validatedlineedit.py`

Содержит `ValidatedLineEdit` — поле ввода с валидацией числового значения.

Класс отвечает за:

- проверку допустимого формата ввода;
- подсветку ошибочного значения;
- отправку сигнала при завершении ввода;
- обработку `Enter` так же, как перехода `Tab`.

`ValidatedLineEdit` не выполняет бизнес-расчёты. Он только сообщает форме, что
ввод завершён и значение можно обработать.

### `functions.py`

Набор вспомогательных функций.

Модуль отвечает за:

- преобразование введённого текста в число;
- форматирование рублей и копеек;
- преобразование чисел в текст через `num2words`;
- вывод суммы в `QLineEdit`;
- копирование текста в буфер обмена;
- показ сообщений пользователю.

Бизнес-состояние приложения здесь не хранится.

### `constants.py`

Хранилище констант приложения:

- путь к файлу интерфейса;
- процент перечисления в корпорацию;
- процент НДС;
- стили полей;
- регулярное выражение для ввода чисел;
- текстовые константы для форматирования сумм.

## Поток данных

1. Пользователь вводит сумму, заплаченную клиентами.
2. Пользователь поочерёдно вводит платежи в корпорацию.
3. `ValidatedLineEdit` сообщает `Report`, что ввод завершён.
4. `Report` обновляет внутренние значения.
5. `Report` выполняет расчёты.
6. `functions.py` форматирует рассчитанные суммы в текст.
7. `Report` выводит текст в поля результата.
8. Пользователь кликает по любому полю результата, и текст копируется в буфер
   обмена.

## Динамические платежи

Первое поле платежа находится в `.ui`-файле.

Когда пользователь вводит сумму платежа `N`, метод `Report.add_payment_field`
создаёт следующую строку:

- метку `Платёж № N+1`;
- поле ввода `edtPaid_N+1`;
- атрибут `paid_N+1`;
- связь сигнала завершения ввода с обработчиком формы.

Так форма не ограничивает пользователя заранее заданным количеством платежей.

## Справочные значения

Справочные значения:

- процент перечисления в корпорацию;
- процент НДС.

Они задаются константами, но могут быть временно изменены в интерфейсе.

Перед применением изменения приложение показывает предупреждение: новое значение
действует только в текущем сеансе. После перезапуска будут восстановлены
значения из `constants.py`.

Если справочное поле очищено или содержит недопустимое значение, приложение
восстанавливает текущее корректное значение.
